# Refresh command palette previews

## Goal

Refresh the README Admin Console preview assets so GitHub visitors see the current polished console, including the new topbar command entry added in the command palette slice.

## Background

The README embeds two tracked screenshots generated by `npm run capture:preview`:

- `docs/assets/admin-console.png` at `1440x960`.
- `docs/assets/admin-console-mobile.png` at `390x844`.

Both screenshots were generated before the command palette was added. The real Admin Console now exposes a visible `命令` topbar action and `Ctrl K / Cmd K` command workflow. The screenshots should stay aligned with the current local demo UI because they are part of the repository's GitHub-facing first impression.

## Requirements

- Use the existing `npm run capture:preview` flow; do not hand-edit binary images.
- Keep screenshot dimensions and paths unchanged.
- Do not add external assets, fonts, UI dependencies, or README copy changes unless the capture flow reveals a broken reference.
- Visually inspect both refreshed PNG files after capture.
- Verify the existing tests that pin preview paths, PNG signatures, dimensions, and minimum file sizes.

## Acceptance Criteria

- [ ] `docs/assets/admin-console.png` is regenerated from the current local demo UI and remains `1440x960` PNG.
- [ ] `docs/assets/admin-console-mobile.png` is regenerated from the current local demo UI and remains `390x844` PNG.
- [ ] The desktop screenshot includes the current topbar utility group with the command action visible.
- [ ] The mobile screenshot remains usable with no obvious clipped topbar, navigation, log, or trace content.
- [ ] `npx vitest run test/demo.test.ts` passes.
- [ ] `npm run verify` passes or any skipped full check is explicitly documented.

## Out of Scope

- Changing the screenshot capture script behavior.
- Adding new screenshots or README sections.
- Runtime Admin Console behavior changes.
