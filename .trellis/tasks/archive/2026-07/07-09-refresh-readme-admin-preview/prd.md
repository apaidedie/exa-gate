# Refresh README admin preview screenshots

## Goal

Refresh the README Admin Console screenshots so GitHub visitors see the current polished UI, including the latest request-log hit target and refresh-feedback improvements.

## Background

- `README.md` embeds `docs/assets/admin-console.png` and `docs/assets/admin-console-mobile.png`.
- The screenshots are generated by `npm run capture:preview`, which launches the local demo server and captures deterministic desktop and mobile views.
- The current screenshot files were last generated before the latest Admin Console UI polish slices.
- `test/demo.test.ts` already pins the screenshot paths, dimensions, PNG signature, and minimum file size.

## Requirements

- Regenerate both README screenshots with the existing `npm run capture:preview` workflow.
- Keep screenshot dimensions unchanged: desktop `1440x960`, mobile `390x844`.
- Do not hand-edit binary images or change README copy unless the capture flow reveals a documentation mismatch.
- Preserve the reproducible capture script and existing demo data flow.
- Verify the refreshed assets remain non-empty and referenced by README/docs/scripts documentation.

## Acceptance Criteria

- [ ] `docs/assets/admin-console.png` is refreshed from the current local demo UI and remains `1440x960` PNG.
- [ ] `docs/assets/admin-console-mobile.png` is refreshed from the current local demo UI and remains `390x844` PNG.
- [ ] `npx vitest run test/demo.test.ts` passes.
- [ ] `npm run verify` passes before archiving.

## Out Of Scope

- Changing Admin Console behavior or layout.
- Changing `scripts/capture-admin-preview.ts` unless the existing capture flow breaks.
- Adding new README sections or marketing copy.
