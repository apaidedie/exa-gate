# Repository Launch Polish

## Goal

Improve the GitHub-facing launch experience so a new visitor can understand the product value, trust the operational/security posture, run the local demo, and choose a deployment path without reading the whole repository.

## Confirmed Facts

- The repository is a static Admin Console plus Fastify/TypeScript reverse proxy; no new frontend framework, external font, CDN, or runtime asset dependency should be introduced.
- README already includes CI, CodeQL, Docker, version, and license trust signals plus three rendered Admin Console screenshots generated by `npm run capture:preview`.
- `npm run demo:ui` starts a realistic local console with demo data and the `admin_local_token` admin token.
- Existing static tests pin README preview image paths, dimensions, non-empty PNG bytes, and key repository hygiene claims.

## Requirements

- Tighten the README first-viewport narrative around who the project is for, what it protects, what it shows, and how quickly it can be tried.
- Add a compact launch/fit section that helps operators decide whether to use the project before they reach the long API tables.
- Keep all claims grounded in current code, scripts, docs, screenshots, and tests; do not advertise unsupported routes, cloud features, SaaS behavior, or unaudited security guarantees.
- Keep visual language polished, simple, and operational rather than marketing-heavy.
- Keep documentation links and docs index aligned with the new launch story.
- Extend tests so future edits preserve the improved GitHub-facing narrative and demo/deployment path.

## Acceptance Criteria

- [x] README has a stronger first-viewport positioning block and concise operator decision support.
- [x] README still links to the local demo, rendered console screenshots, Docker deployment, management API, and security model.
- [x] Docs index points readers to the new or updated launch/deployment material without duplicating the full README.
- [x] Static tests verify the new README launch claims, demo route, screenshot evidence, and docs index language.
- [x] `npx vitest run test/demo.test.ts test/project-hygiene.test.ts`, `git diff --check`, and `npm run verify` pass before the task is committed.

## Out Of Scope

- No backend route changes.
- No Admin Console runtime UI change in this task.
- No generated screenshots unless the README references or screenshot evidence changes.
- No package dependency additions.
